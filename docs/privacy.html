隐私政策

MarkMate 是一款本地优先的编辑器。你的数据由你做主。

1. 我们的原则

MarkMate 的设计核心理念是本地优先。你的 Markdown 文档、JSON 文件、配置数据等，全部存储在你的计算机本地文件系统中。MarkMate 不会将你的文档内容上传到任何远程服务器。

2. 我们收集什么

2.1 文档内容

我们不收集、不上传、不存储你的任何文档内容。所有文件读写操作仅在你的本地文件系统进行。MarkMate 没有后端服务器来接收文档数据。

2.2 匿名使用统计（计划中，尚未启用）

未来版本中，MarkMate 可能会在获得你明确同意的情况下，匿名上报聚合使用数据，以帮助我们改进产品。这些数据仅包括：

  • 应用启动次数
  • 打开的文件类型分布（如 .md 占比多少、.json 占比多少）
  • 导出功能使用次数（PDF/HTML/Word 各多少）
  • 崩溃场景与错误类型
  • 使用的操作系统与 MarkMate 版本

我们不会收集：

  • 任何文档内容
  • 文件名或文件路径
  • 个人身份信息
  • IP 地址或其他网络标识

此功能将在首次使用时弹出明确提示，你可以选择同意或拒绝。拒绝后不会有任何数据上报。你可以在设置中随时开启或关闭。

2.3 下载页面统计（仅网站）

本站点（GitHub Pages）使用浏览器本地存储（localStorage）记录页面访问与下载点击行为。这些数据仅存储在你的浏览器中，不会被发送到任何服务器。我们使用这些数据来了解哪个版本的软件更受欢迎，以便优化下载体验。

3. 第三方服务

MarkMate 应用本身不集成任何第三方分析、广告或追踪 SDK。MarkMate 网站托管在 GitHub Pages 上，GitHub 可能会按照其隐私声明收集访问日志。

4. 数据安全

由于你的文档完全存储在本地，数据安全取决于你的计算机安全实践。我们建议：

  • 定期备份重要文档
  • 使用操作系统提供的全盘加密功能（如 macOS FileVault、Windows BitLocker）
  • 不要在不信任的计算机上打开敏感文档

5. 开源透明

MarkMate 是完全开源的（MIT 协议）。所有源代码公开在 GitHub。你可以自行审查代码，确认没有隐藏的数据收集行为。欢迎提交 Issue 或 PR 对隐私实践提出改进建议。

6. 联系我们

如果你对隐私政策有任何疑问或建议，请通过以下方式联系：

  • 提交 GitHub Issue
  • 填写反馈问卷

7. 政策更新

本隐私政策可能会随产品迭代而更新。重大变更将通过 GitHub Release Notes 和本页面公告。
